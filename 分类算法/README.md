# 乳腺癌早筛辅助分诊系统

本项目是一个面向机器学习课程大作业的真实场景化演示系统。它不再只是“训练三个分类模型然后展示指标”，而是把模型放进一个更贴近实际应用的业务场景中，做成了一个可以直接操作的 **乳腺癌早筛辅助分诊系统**。

简单理解，这个系统要解决的问题是：

> 当我们拿到一位患者的检查特征后，能不能用机器学习模型快速判断她更偏向良性还是恶性，并给出风险等级、分诊建议，以及批量筛查时应该优先处理哪些样本。

系统使用公开数据集 `sklearn.datasets.load_breast_cancer`，训练了三种分类模型：

1. `Logistic Regression`
2. `Random Forest`
3. `SVM`

然后通过 Streamlit 做成了一个可视化系统，支持：

- 单例分诊
- 批量筛查
- 风险分层
- 模型对比
- 结果解释
- 图表展示

---

## 1. 先用一句人话解释这个系统是干什么的

如果你对机器学习完全不熟，可以把这个系统理解成一个“**辅助判断工具**”：

- 你输入一组患者检查指标
- 系统会调用训练好的模型
- 模型输出“更像良性还是更像恶性”
- 系统再把这个结果翻译成更好理解的话：
  - 低风险
  - 中风险
  - 高风险
- 同时给出建议，例如：
  - 常规随访
  - 建议短期复查
  - 建议尽快进一步检查

它的意义不在于替代医生，而在于：

- 帮助快速筛查
- 帮助排序优先级
- 帮助展示机器学习模型如何真正服务一个现实问题

**注意：本系统仅用于教学演示，不替代医生诊断。**

系统主入口。

依赖清单。  
如果在新环境中运行项目，通常先执行：

```bash
pip install -r requirements.txt
```

存放数据集或中间数据文件。

存放训练好的模型文件和模型元信息，例如：

## 6. 左侧“筛查配置”是什么意思

页面左侧有一个侧边栏，里面是整个系统的主要配置项。

### 6.1 主模型

这里可以选择当前系统主要采用哪个模型给出分诊结论。

可选项有：

- Logistic Regression
- Random Forest
- SVM

如果你不确定选哪个，默认推荐使用 **Logistic Regression**，因为当前训练结果下它的 Recall、AUC 和 Accuracy 都表现很好，而且解释性也更强。

### 6.2 开启三模型对比

勾选后，系统不仅会显示主模型结果，还会把三个模型的结果一起展示出来。

这非常适合答辩，因为你可以清楚展示：

- 三个模型是否得出相同结论
- 哪些样本存在模型分歧
- 为什么最后推荐其中一个模型作为主模型

如果你只是想让页面更简洁，可以关闭它，这样系统只显示主模型。

### 6.3 中风险阈值

这个值决定从什么概率开始，把样本划为“中风险”。

默认值是：

```text
0.35
```

意思是：

- 如果恶性概率低于 0.35，通常判为低风险
- 如果恶性概率在 0.35 到高风险阈值之间，判为中风险

### 6.4 高风险阈值

这个值决定从什么概率开始，把样本划为“高风险”。

默认值是：

```text
0.70
```

意思是：

- 恶性概率达到或超过 0.70，就进入高风险
- 这类样本应该优先安排复查或进一步检查

### 6.5 解释区展示特征数

这个值控制系统在“解释区”展示多少个关键特征。

默认值通常是：

```text
6
```

也就是系统会挑出最值得关注的 6 个特征，帮助使用者快速理解：

- 为什么系统会判成高风险
- 哪些特征最异常
- 哪些特征最值得医生重点复核

## 9. 单例分诊怎么用

这是最像“真实业务使用”的页面。

### 它是干什么的

用于输入 **一个病例** 的特征值，然后立即得到：

- 预测类别
- 恶性概率
- 良性概率
- 风险等级
- 分诊建议
- 模型解释

### 页面结构怎么理解

这个页面会把特征分成三组：

#### 1. 平均值特征

例如：

- `mean radius`
- `mean texture`
- `mean perimeter`

这些可以理解为某些测量指标的平均水平。

#### 2. 误差特征

例如：

- `radius error`
- `texture error`

这些可以理解为测量中的变化或误差量。

#### 3. 最差值特征

例如：

- `worst radius`
- `worst texture`

这些可以理解为样本中最明显或最极端的数值表现。

### 如果我完全不懂这些特征怎么办

不用慌。  
系统已经给每个输入框填好了默认值，这些默认值来自数据集的中位数。

你可以这样理解：

- 默认值代表“一个比较中间、比较常见”的水平
- 你只需要适当改动部分特征，就能看到预测结果如何变化

### 使用步骤

1. 在“病例编号”里输入一个编号，例如 `CASE-001`
2. 保持默认值，或者修改部分特征
3. 点击 `生成分诊结论`
4. 查看结果

### 结果出来后怎么看

系统会先显示一个醒目的结论卡片：

- 低风险
- 中风险
- 高风险

然后显示：

- 主模型是谁
- 恶性概率是多少
- 良性概率是多少
- 预测类别是什么

如果开启了三模型对比，还会继续显示：

- 三个模型分别怎么判断
- 三个模型是否一致
- 是否存在模型分歧

### 分诊建议是什么意思

系统会根据风险等级给出直观建议：

#### 低风险

建议常规随访。  
意思是当前模型认为恶性倾向不高，但仍然应该结合医生意见继续观察。

#### 中风险

建议短期复查。  
意思是当前模型发现一定风险，但还没有高到最紧急的程度。

#### 高风险

建议尽快进一步检查。  
意思是恶性概率较高，应该优先处理。

### 模型解释区是干什么的

这是系统非常重要的一部分。

它不是只告诉你“结果是什么”，还会告诉你“为什么会这样判断”。

不同主模型下，解释方式略有不同：

#### 如果主模型是 Logistic Regression

系统会展示：

- 哪些特征提高了恶性风险
- 哪些特征降低了恶性风险
- 每个关键特征的贡献大小

这是最容易讲清楚的解释方式，很适合答辩。

#### 如果主模型是 Random Forest

系统会结合：

- 特征重要性
- 当前样本相对中位数的偏离程度

来告诉你哪些特征最值得重点关注。

#### 如果主模型是 SVM

SVM 原生不太容易直接解释，所以系统会借助其它模型的风险权重，做一个“辅助解释视图”。

你可以在答辩中这样说：

> SVM 的预测效果不错，但可解释性较弱，因此系统额外提供了辅助解释方式，帮助使用者理解关键风险因素。

## 10. 批量筛查怎么用

这个页面是整个系统最像“真实工作流程”的部分。

### 它的作用

用于一次性上传多个病例，让系统自动完成：

- 批量预测
- 风险等级划分
- 高风险样本排序
- 输出优先处理队列

### 适合什么场景

比如一天有很多待筛查样本时，人工一个个看会很慢，这时就可以：

1. 上传一个 CSV
2. 让系统自动全部判断
3. 把恶性概率高的样本排在前面
4. 先处理高风险样本

### 怎么操作

#### 第一步：下载示例 CSV

页面里有一个“下载示例 CSV”按钮。  
第一次使用时，建议先下载它看看格式。

这个文件的作用是告诉你：

- CSV 应该长什么样
- 需要哪些列
- 列名必须怎么写

#### 第二步：准备你的 CSV

上传的 CSV 必须包含训练时使用的 30 个特征字段。  
字段名必须一致，否则系统会报错。

你还可以额外加一些不参与建模、但方便识别样本的字段，例如：

- `patient_id`
- `case_id`
- `sample_id`
- `id`
- `患者编号`
- `编号`

这些字段不会影响预测，但会保留在结果里，方便你知道每一行是谁。

#### 第三步：上传 CSV

上传后系统会自动检查：

1. 是否缺少必要字段
2. 是否有无法转换为数值的内容
3. 是否存在额外字段

#### 第四步：查看批量筛查结果

系统会生成一个“优先处理队列”，并按照 **恶性概率从高到低排序**。

也就是说，最上面的样本是最应该优先看的。

### 批量筛查结果里每一列是什么意思

#### 案例编号

样本的编号。  
如果你上传的 CSV 里有 `patient_id`、`case_id` 之类字段，系统会优先使用它们。

#### 模型

当前显示结果来自哪个主模型。

#### 预测类别

模型判断这个样本更偏向：

- `malignant`：恶性
- `benign`：良性

#### 恶性概率

模型认为这个样本属于“恶性”的概率。

#### 良性概率

模型认为这个样本属于“良性”的概率。

#### 风险等级

根据你设置的风险阈值自动转换成：

- 低风险
- 中风险
- 高风险

#### 分诊建议

系统根据风险等级自动生成的处理建议。

### 模型分歧样本是什么

如果开启三模型对比，系统还会额外找出：

> 哪些样本三个模型没有给出一致结论

这类样本非常值得关注，因为它们通常意味着：

- 样本边界模糊
- 特征表现复杂
- 模型判断不稳定

在答辩里你可以把这部分解释为：

> 系统不仅给出结果，还能识别高不确定性样本，提醒人工重点复核。

### 如何下载结果

页面支持直接下载批量筛查结果 CSV。  
这样你可以把结果拿去做：

- 报告表格
- PPT 截图
- 后续人工分析

## 11. 模型中心怎么看

这个页面主要面向：

- 老师
- 答辩评委
- 想看算法效果的人

### 它的作用

用来展示：

- 三种模型的客观指标
- 当前推荐部署模型
- 各种图表结果

### 页面里会展示什么

#### 1. 推荐部署模型

系统会根据：

- Recall
- AUC
- Accuracy

综合排序，给出默认推荐模型。

当前推荐模型通常是：

```text
Logistic Regression
```

### 为什么不是只看 Accuracy

因为在乳腺癌早筛场景里，最可怕的不是“误报一点”，而是“把高风险样本漏掉”。

所以：

- Recall 越高，漏诊越少
- AUC 越高，整体区分能力越强

这也是系统为什么强调筛查场景下要优先看 Recall。

#### 2. 指标表

会展示每个模型的：

- Accuracy
- Precision
- Recall
- F1-score
- AUC

#### 3. 图表结果

页面会展示训练阶段已经生成好的图表，包括：

- 指标对比图
- ROC 曲线
- Random Forest 特征重要性图
- Logistic Regression 系数图
- 三个模型各自的混淆矩阵

这部分非常适合直接拿来作为答辩展示内容。

## 13. 这个系统里的“风险等级”到底是什么意思

很多人第一次用系统时，最容易困惑的是：

> “恶性概率”和“风险等级”有什么区别？

可以这样理解：

### 恶性概率

这是模型的原始输出。  
比如：

```text
恶性概率 = 0.82
```

表示模型认为这个样本有 82% 的可能更偏向恶性。

### 风险等级

这是系统为了让人更容易理解，把概率翻译成了分层结果。

例如默认阈值下：

- 小于 0.35：低风险
- 0.35 到 0.70：中风险
- 大于等于 0.70：高风险

所以你可以把“风险等级”理解成：

> 系统帮你把难懂的概率，翻译成更容易做决策的业务语言。

## 18. 常见问题

### Q1：为什么我上传 CSV 后提示缺少字段？

因为上传文件必须包含训练时用到的 30 个特征字段，而且字段名必须一致。

最简单的做法是：

1. 先下载示例 CSV
2. 按示例格式准备数据
3. 再上传

### Q2：为什么系统里会有“恶性概率”和“良性概率”两个值？

因为二分类模型会给出两个类别的概率，它们加起来大约等于 1。

### Q3：为什么模型结果有时会不一致？

因为不同模型的建模方式不同：

- Logistic Regression 偏线性
- Random Forest 偏树模型集成
- SVM 偏边界分类

当样本接近分类边界时，不同模型给出不同判断是正常的。

### Q4：为什么默认推荐 Logistic Regression？

因为当前训练结果下，它不仅指标表现好，而且更容易解释，适合医疗筛查这种需要“看得懂”的场景。

### Q5：这个系统能直接用于真实医院吗？

不能。  
这是教学演示系统，只能用于课程项目、学习和展示，不是经过医疗认证的产品。
